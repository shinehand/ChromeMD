# ChromeMD Worklog

## 2026-03-29

### 작업
- 프로젝트 구조 및 동작 방식 1차 파악
- 저장 기능을 `바로 저장`과 `다른 이름으로 저장`으로 분리
- 직접 저장 버튼을 일단 숨기고 `다른 이름` 저장만 남김

### 확인한 내용
- 빌드 단계 없이 루트 파일들을 그대로 로드하는 Chrome Extension Manifest V3 프로젝트다.
- 핵심 구현은 `content.js`에 집중되어 있고, `background.js`는 로컬 디렉터리 탐색과 팝업 창 열기 보조 역할을 맡는다.
- `manifest.json` 기준 주요 권한은 `downloads`, `tabs`, `scripting`, `windows`, `storage`이며 `file:///*` 접근을 사용한다.
- 마크다운 파싱은 `marked`, 코드 하이라이팅은 `highlight.js`를 번들된 정적 파일로 직접 포함한다.
- 옵션 페이지는 `options.html` + `options.js` 조합으로 `autoPopup` 설정 하나를 저장한다.

### 기능 요약
- 로컬/원격 마크다운 URL 감지 후 페이지를 자체 UI로 재구성
- 보기, 편집, 분할 모드 전환
- 저장되지 않은 변경 표시
- 로컬 파일 바로 저장
- 다른 이름으로 저장
- 코드 블록 복사 버튼
- TOC 생성 및 활성 헤딩 추적
- 로컬 `file://` 환경에서 폴더 파일 탐색
- 로컬 파일 자동 팝업 열기 옵션

### 이번 변경 메모
- 툴바 저장 버튼을 `저장`, `다른 이름` 두 개로 분리
- `Ctrl/Cmd+S`는 `저장` 동작에 연결
- 로컬 `file://` 파일은 File System Access API와 서비스워커 IndexedDB를 이용해 현재 파일 핸들을 기억하고 다음 저장부터 바로 덮어쓴다
- macOS에서 현재 파일 경로로 바로 저장하기 위해 Native Messaging 기반 네이티브 호스트 경로를 추가했다
- 브라우저가 직접 파일 쓰기를 지원하지 않는 경우 기존 다운로드 저장으로 폴백한다
- 자동 검증을 위해 data attribute 기반 테스트 전용 저장 브리지 훅을 추가했다
- `저장` 버튼은 수정 전에는 회색 비활성 상태로 두고, 수정이 생기면 활성 색상으로 바뀌게 했다
- 파일 핸들 저장 위치를 콘텐츠 스크립트 쪽이 아니라 서비스워커 쪽으로 옮겨 파일별 재사용 안정성을 높였다
- 저장된 파일 핸들의 권한 상태가 `prompt`여도 유지하도록 바꿔, 페이지 재진입 때 핸들을 너무 일찍 버리지 않게 수정했다
- 옵션 페이지에 확장 ID와 네이티브 헬퍼 상태 표시를 추가했다
- 직접 저장 기능은 나중에 다시 이어가기 위해 UI와 단축키에서만 잠시 숨겼다
- 커밋 대상만 남기기 위해 `output/`, `__pycache__/`, `*.pyc`, `.DS_Store`를 `.gitignore`에 추가했다

### 검증 결과
- 브라우저 자동화로 `저장` 수행 시 현재 파일 내용이 갱신되는 것을 확인
- 브라우저 자동화로 `다른 이름` 수행 시 별도 파일이 생성되는 것을 확인
- `저장` 후 dirty 표시가 사라지는 것을 확인
- `다른 이름` 후에는 현재 문서 기준 dirty 표시가 유지되는 것을 확인
- 브라우저 자동화로 초기 `저장` 버튼이 비활성, 수정 후 활성, 직접 저장 후 다시 비활성으로 돌아가는 상태 변화를 확인
- 네이티브 호스트 프로토콜 `status`, `writeFile` 호출이 정상 응답하고 실제 파일 쓰기가 되는 것을 확인
- 네이티브 호스트 설치 스크립트가 Chrome Native Messaging manifest를 올바른 형식으로 생성하는 것을 확인

### 파일별 메모
- `manifest.json`: 확장 진입 설정과 content script 매칭 패턴 정의
- `content.js`: 렌더링, 편집, 저장, TOC, 파일 탐색기, 상태바, 단축키까지 대부분 담당
- `background.js`: 디렉터리 listing 추출과 새 팝업 창 생성 담당
- `styles.css`: GitHub 스타일 기반 UI와 라이트/다크 모드 스타일 정의
- `options.html`, `options.js`: 자동 팝업 설정 UI
